# PDF 데이터 추출 시스템 계획

## 1. 개요
- 사용자는 PDF 파일을 업로드하여 그 안의 데이터를 추출하고 테이블 형식으로 확인할 수 있는 웹 기반 시스템을 개발한다.
- 시스템은 PDF와 함께 전송할 추가 프롬프트를 관리할 수 있으며, 무료 AI 모델을 활용하여 PDF 데이터를 추출한다.

## 2. 화면 구성
### 2.1 상단 영역: 파일 첨부
- PDF 파일 업로드 드롭존 및 파일 선택 버튼
- 업로드 가능한 파일 형식: PDF, 이미지(JPG/PNG 등)
- 업로드 상태 표시: 업로드 진행률, 성공/실패 메시지

### 2.2 우측 상단: 설정 버튼
- 설정 버튼 클릭 시 팝업 또는 사이드 패널 노출
- 설정 기능
  - PDF 전송 시 함께 보낼 기본 프롬프트 입력
 
### 2.3 하단 영역: 데이터 표시 테이블
- 추출된 데이터를 표 형태로 표시
- 컬럼 예시: 날짜, 현장명, 공급자, 품목, 규격, 단위, 물량, 단가, 금액, 세액, 합계, 통화
- 테이블 정렬, 검색, 필터 기능
- CSV/Excel 다운로드 버튼
- 원본 PDF 다시 열기, 재추출 버튼

## 3. 주요 기능
### 3.1 PDF 데이터 추출
- 무료 AI 모델 연동을 통한 PDF 데이터 추출
- PDF 파일에서 텍스트와 표를 판독하여 구조화된 데이터로 변환
- 이미지 기반 PDF의 경우 OCR 처리 후 AI 모델로 데이터 해석
- 추출 결과를 시스템 내부 포맷으로 변환하여 테이블에 표시

### 3.2 프롬프트 관리
- 설정 메뉴에서 PDF와 함께 전송할 프롬프트를 작성/저장
- 프롬프트 예시
  - "이 PDF에서 견적서 항목을 추출하여 날짜, 공급자, 품목, 단가, 금액을 표로 정리해줘"
  - "PDF 속 표 데이터를 CSV로 변환해줘"
- 저장된 프롬프트를 기본값으로 유지하거나 업로드 시 선택해서 전송
- 프롬프트 버전 관리 또는 최근 사용 프롬프트 이력 표시

### 3.3 무료 AI 모델 통합
- 무료로 사용 가능한 AI 추출 모델과의 연동 방식 설계
- 외부 API 호출 시 PDF 파일 및 추가 프롬프트를 함께 전송
- 응답 결과를 받아서 데이터 검증 및 후처리
- 모델 응답 실패 시 재시도 또는 대체 처리

## 4. 기술 구성 제안
### 4.1 프론트엔드
- React / Vue / Svelte 등 SPA 프레임워크
- 파일 업로드 컴포넌트
- 테이블 UI: 정렬, 필터, 페이지네이션
- 설정/프롬프트 관리 UI

### 4.2 백엔드
- 파일 수신 및 AI 모델 요청 처리 API
- PDF/이미지 업로드 저장 및 임시 보관
- AI 모델 호출 및 결과 파싱 엔드포인트
- 추출 데이터 검증 및 반환

### 4.3 AI 연동
- 무료 AI 모델 제공 서비스 선택
- PDF 파일 또는 텍스트 추출본을 전송하는 API 설계
- 프롬프트와 함께 모델 요청을 구성
- 응답 데이터를 테이블 필드로 매핑

### 4.4 DB 구성
- 추출된 데이터, 업로드된 파일 메타데이터, 프롬프트 설정을 영구 저장
- 관계형 DB(MySQL, PostgreSQL) 또는 NoSQL(MongoDB) 중 요구에 맞게 선택
- 배치 및 검색 성능을 고려한 인덱스 설계

#### 4.4.1 주요 테이블/컬렉션
- `Users`(선택적)
  - `user_id`, `name`, `email`, `created_at`
- `Documents`
  - `document_id`, `user_id`, `file_name`, `file_type`, `uploaded_at`, `status`, `source_url`
- `Prompts`
  - `prompt_id`, `user_id`, `name`, `content`, `created_at`, `updated_at`
- `ExtractedRecords`
  - `record_id`, `document_id`, `row_index`, `date`, `location`, `supplier`, `item`, `spec`, `unit`, `quantity`, `unit_price`, `amount`, `tax`, `total`, `currency`, `raw_data`, `created_at`
- `ExtractionHistory`
  - `history_id`, `document_id`, `prompt_id`, `status`, `requested_at`, `completed_at`, `error_message`

#### 4.4.2 저장 전략
- 문서 메타데이터는 `Documents`에 저장하고 파일은 파일 스토리지(S3, Azure Blob, 로컬 저장소)에 보관
- 추출 결과를 `ExtractedRecords`에 행 단위로 저장하여 테이블 조회 및 다운로드 지원
- 프롬프트는 `Prompts`에 저장해 재사용 및 편집 가능
- 처리 이력은 `ExtractionHistory`에 남겨서 재추적 및 오류 분석

5. 추출 결과 테이블 표시 및 다운로드 기능 추가
6. 설정/프롬프트 관리 기능 완성
7. 테스트 및 개선

## 6. 기대 효과
- 사용자가 별도 툴 없이 PDF에서 구조화된 데이터를 즉시 가져올 수 있다.
- 프롬프트 관리 기능으로 일관된 AI 추출 요청이 가능하다.
- 무료 AI 모델 연동으로 초기 비용 부담을 최소화한다.
